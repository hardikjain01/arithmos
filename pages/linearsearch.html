Linear Search

  • Start from the leftmost element of arr[] and one by one compare x with each element of arr[]
  • If x matches with an element, return the index.
  • If x doesnâ€™t match with any of elements, return -1.
back 
                                    
package algorithms;

import java.util.Random;

/** A class to implement Linear Search
 * 
 * @author Hardik Jain
 *
 */

public class LinearSearch {
	private int[] data;
	private int size;
	
	public LinearSearch(int[] data) {
		this.data = data;
		this.size = this.data.length;
	}
	
	/**implements linear search
	 * 
	 * @param element to be searched
	 * @return index of element in data if found else -1
	 */
	public int search(int element) {
		// Searching for the element at every index of array
		for(int i = 0; i<size; i++) {
			if(data[i] == element) {
				return i;
			}
		}
		// If element is not found in the array
		return -1;
	}

	/**
	 * @return the size
	 */
	public int getSize() {
		return size;
	}
	
	/** 
	 * To print the array
	 */
	public void print() {
		System.out.print("[ ");
		for(int i=0; i<size; i++) {
			System.out.print(data[i]);
			if(i<size-1) {
				System.out.print(", ");
			}
		}
		System.out.print(" ]");
	}
	
	public static void main(String[] args) {
		
		/*
		 * generates random numbers between 0-30
		 * and stores them in the array
		 */
		Random rand = new Random();
	    int[] array = new int[20];
	    for(int i=0;i<20;i++) {
	    	array[i] = rand.nextInt(30);
	    }
	    
	    int toFind = 23;
	    LinearSearch LS = new LinearSearch(array);
	    LS.print();
	    int index = LS.search(toFind);
	    if(index == -1) {
	    	System.out.println("\nElement not found.");
	    }
	    else {
	    	System.out.println("\n" + toFind + " found at index " + index);
	    }
	}
	
}

                                    
                                    
Output:

                                    
[ 24, 26, 4, 19, 17, 16, 23, 0, 14, 3, 29, 14, 16, 7, 15, 29, 28, 28, 26, 0 ]
23 found at index 6

The time complexity of above algorithm is O(n).

Linear search is rarely used practically because other search algorithms such as the binary search algorithm and hash tables allow significantly faster searching comparison to Linear search.
